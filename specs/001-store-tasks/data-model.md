# Data Model: Store Tasks

Entity: Task

- `id` (string): Unique identifier (UUID4). Required.
- `title` (string): Short title for the task. Required, non-empty, max length 1024.
- `description` (string|null): Optional longer description.
- `status` (string): One of `open` or `completed`. Default: `open`.
- `created_at` (string, ISO-8601): Timestamp when the task was created.
- `updated_at` (string, ISO-8601|null): Timestamp when the task was last updated.
- `due_date` (string|null): Optional ISO-8601 date/time when task is due.
- `priority` (string|null): Optional, e.g., `low`, `medium`, `high`.

Validation rules:

- `title` must be present and trimmed; reject empty or whitespace-only titles.
- `status` must be `open` or `completed`.
- `id` must be a valid UUID string generated by implementation.

Storage layout (JSON document):

Root object:

{
  "tasks": [ { Task }, { Task }, ... ]
}

Operations:

- Create: validate fields, assign `id`, set `created_at` and `updated_at`.
- Update: find by `id`, apply allowed updates (`title`, `description`, `due_date`, `priority`, `status`), set `updated_at`.
- Delete: remove by `id`.
- List: return tasks, optionally filtered by `status` or searchtext.

Concurrency assumption: single-writer (single-user local). Implementation uses atomic replace on write to avoid partial writes.
